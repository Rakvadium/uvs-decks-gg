## Cursor Cloud specific instructions

### Project overview
TCGDecks.GG — a Next.js 16 (App Router) + React 19 TCG deck builder for the UniVersus card game. Convex is the sole serverless backend (data, auth, real-time). No Docker or separate database required.

### Key commands
| Task | Command |
|------|---------|
| Install deps | `npm install` |
| Dev server | `npm run dev` (port 8090) |
| Lint | `npm run lint` |
| Build | `npm run build` |

### Environment variables
The app requires `NEXT_PUBLIC_CONVEX_URL` to connect to Convex. Without it, the Next.js shell renders but data-dependent pages show loading states. For full functionality, the Convex deployment must be configured with `CONVEX_DEPLOYMENT` as well.

To run `npx convex dev` (which syncs schema/functions to a dev deployment), both `CONVEX_DEPLOYMENT` and authentication with Convex are needed.

### Caveats
- ESLint exits with code 1 due to pre-existing warnings/errors in the codebase; this is normal.
- The `convex/_generated/` directory is auto-generated by `npx convex dev`; do not edit files there manually.
- Admin import scripts (`npm run import:universus`, `npm run bulk-upload`, etc.) require `bun` runtime, which is not part of the standard dev setup.
- The Community page renders fully with static/mock content; Gallery and Decks pages require a live Convex backend for data.
